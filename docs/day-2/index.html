0. 환경 설정 개요 및 주의사항

본 페이지는 Day 2 본 실습을 진행하기 전, 반드시 완료되어야 하는 운영체제 및 미들웨어(ROS2) 설정을 다룹니다.

💡
설치 방식 관련 안내 (Dual-Boot vs WSL):
ROS2를 가장 안정적이고 높은 성능으로 사용하기 위해서는 Ubuntu 듀얼 부팅 환경을 권장합니다. 하지만 하드웨어 리소스 문제나 리눅스 사용 경험이 적어 듀얼 부팅 세팅이 부담스러운 학생들을 위해, 본 문서에서는 윈도우 환경에서 WSL(Windows Subsystem for Linux)을 활용하여 손쉽게 개발 환경을 구축하는 방식을 중점적으로 안내합니다.
1단계 ─ 운영체제 및 미들웨어 세팅

1. Windows 11 환경에서 WSL 2 및 ROS 2 Humble 설치 매뉴얼

Windows 11 환경에서 WSL 2를 이용해 ROS 2 Humble을 설치하고, GUI 환경(Gazebo, Rviz2)을 구성하는 방법을 단계별로 상세히 설명합니다.

📖 이론 설명: WSL과 GUI 연동 원리

WSL은 기본적으로 터미널(CLI) 환경만 제공합니다. 로봇 시각화(Rviz2)나 3D 시뮬레이션(Gazebo)을 실행하기 위해서는 화면에 창을 띄워줄 그래픽 서버가 필요합니다. 이를 위해 Windows 쪽에 X 서버 프로그램(VcXsrv)을 띄워두고, WSL 내부의 리눅스에서 발생하는 그래픽 출력 데이터를 Windows로 전달(export DISPLAY)하여 화면에 렌더링하는 방식을 사용합니다.

📌 주요 환경설정 레퍼런스

명령어 / 설정값 역할 설명
wsl --install WSL 기본 설치 Windows Subsystem for Linux를 활성화하고 기본 배포판을 설치합니다.
export DISPLAY=... 그래픽 출력 우회 설정 WSL2 내부의 그래픽을 Windows(호스트)의 X 서버로 전달하기 위한 통신 설정입니다.

⚙️ 단계별 설치 매뉴얼

1
WSL 2 및 Ubuntu 22.04 설치

가장 먼저 Windows 내에서 리눅스를 실행할 수 있는 환경을 만들어야 합니다.

  • PowerShell 관리자 권한 실행: 시작 메뉴에서 'PowerShell'을 검색한 뒤, 반드시 '관리자로 실행'을 클릭하세요.
  • 가상화 기능 활성화: 하단 명령어 모음의 1단계 명령어를 실행합니다.
  • WSL 설치 및 버전 설정: wsl --install 및 wsl --set-default-version 2 입력 후 컴퓨터를 재시작합니다.
  • Ubuntu 22.04 설치: 재시작 후 터미널에 wsl --install -d Ubuntu-22.04를 입력합니다. 설치가 완료되면 사용자 이름(username)과 비밀번호를 설정합니다. (비밀번호 입력 시 화면에 안 보여도 정상입니다.)
2
Windows Terminal 설치 (권장)

여러 개의 터미널 탭을 효율적으로 관리하기 위해 설치하는 것이 좋습니다.

  • Microsoft Store에서 'Windows Terminal'을 검색하여 설치합니다.
  • 설정에서 기본 프로필을 'Ubuntu 22.04'로 변경하면 터미널을 열 때마다 바로 리눅스 환경으로 접속할 수 있습니다.
3
ROS 2 Humble 설치

설치된 Ubuntu 터미널을 열고 본격적인 ROS 2 설치를 진행합니다.

  • 언어(Locale) 설정: UTF-8을 지원하도록 설정합니다.
  • 저장소 및 GPG 키 추가: ROS2 패키지를 다운받을 수 있도록 공식 저장소 경로를 추가합니다.
  • ROS 2 데스크탑 버전 설치: 시뮬레이션에 필요한 GUI 툴들이 포함된 ros-humble-desktop을 설치합니다.

※ 실행 명령어는 하단의 '단계별 터미널 명령어 모아보기'를 참고하여 순서대로 복사/붙여넣기 하세요.

4
시뮬레이션 및 GUI 설정 (Gazebo & Rviz2)

WSL에서 GUI 화면을 띄우기 위해서는 추가 그래픽 서버 설정이 필요합니다.

  • VcXsrv 설치: Windows용 X 서버 프로그램을 다운로드하여 설치합니다.
  • XLaunch 실행: 설치 후 XLaunch를 실행하여 Multiple windows ➔ Start no client ➔ Disable access control (체크 필수!) 옵션으로 실행합니다.
  • Gazebo 설치: 터미널에서 하단의 4단계 스크립트를 실행하여 시뮬레이터를 설치합니다.
  • 방화벽 허용: 창이 뜨지 않는다면 Windows 방화벽 설정에서 VcXsrv의 통신을 모두 허용해야 합니다.
5
환경 설정 자동화 및 정상 작동 확인

터미널을 켤 때마다 ROS2를 수동으로 활성화하지 않도록 설정을 자동화합니다.

  • gedit ~/.bashrc 명령어로 설정 파일을 엽니다.
  • 파일 맨 아래에 ROS2 소싱 및 Display 통신 우회 설정을 추가하고 저장합니다.
  • 통신 테스트: 터미널 두 개를 열어 한 쪽엔 talker, 다른 쪽엔 listener 노드를 실행해 메시지 송수신을 확인합니다.
  • 시각화 툴 테스트: 터미널에 rviz2를 입력하여 그래픽 창이 뜨는지 확인합니다.

✨ 예상 실행 결과

💻 단계별 터미널 명령어 모아보기 (복사용) 
# [1단계] PowerShell (관리자 권한)
dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart
dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart
wsl --install
wsl --set-default-version 2

# (컴퓨터 재시작 후) PowerShell
wsl --install -d Ubuntu-22.04
# [3단계] Ubuntu 터미널: 언어(Locale) 설정
sudo apt update && sudo apt install locales
sudo locale-gen en_US en_US.UTF-8
sudo update-locale LC_ALL=en_US.UTF-8 LANG=en_US.UTF-8
export LANG=en_US.UTF-8

# [3단계] Ubuntu 터미널: 저장소 및 GPG 키 추가
sudo apt install software-properties-common
sudo add-apt-repository universe
sudo apt update && sudo apt install curl -y
sudo curl -sSL https://raw.githubusercontent.com/ros/rosdistro/master/ros.key -o /usr/share/keyrings/ros-archive-keyring.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/ros-archive-keyring.gpg] http://packages.ros.org/ros2/ubuntu $(. /etc/os-release && echo $UBUNTU_CODENAME) main" | sudo tee /etc/apt/sources.list.d/ros2.list > /dev/null

# [3단계] Ubuntu 터미널: ROS 2 Desktop 설치
sudo apt update
sudo apt upgrade
sudo apt install ros-humble-desktop
# [4단계] Ubuntu 터미널: Gazebo 시뮬레이터 설치
curl -sSL http://get.gazebosim.org | sh

2. 두산 로봇 패키지 설치

ROS2 환경 구성이 끝났다면, 이번 실습에서 사용할 두산 협동 로봇(Doosan Robotics) ROS2 패키지를 다운로드하고 빌드하여 시스템에 인식시켜야 합니다.

DoosanRobotics/doosan-robot2

ROS2 packages for Doosan collaborative robots. This repository provides ROS2 support for Doosan robots, including descriptions and simulation components.

github.com

📖 이론 설명: ROS2 워크스페이스 구조

ROS2 패키지를 소스 코드로 설치할 때는 전용 작업 공간인 Colcon Workspace(일반적으로 ~/ros2_ws)를 만들고, 그 안의 src 폴더에 깃허브 코드를 다운로드(Clone) 합니다. 이후 루트 디렉토리에서 colcon build 명령어를 통해 코드를 컴파일하면, 윈도우의 .exe 파일이 만들어지듯 ROS2에서 실행 가능한 노드와 모델 파일들이 생성됩니다.

📌 주요 명령어 레퍼런스

명령어 역할 설명
mkdir -p 디렉토리 생성 -p 옵션으로 상위 경로까지 한 번에 생성
git clone 소스 코드 다운로드 원격 저장소의 코드를 로컬 환경으로 복사
rosdep install 의존성 패키지 설치 다운받은 패키지가 필요로 하는 외부 라이브러리 자동 설치
colcon build 패키지 빌드 워크스페이스 내의 모든 소스 코드를 컴파일

source install/setup.bash
